4. The Bug: Sam's Failed Chat Send

Discovery

At 06:07:32, Sam's first checkWork invocation reported: "Team is mobilized" with delegation messages supposedly sent to Jules, Noor, Rio, and Tess.

However, inspection of the Chat database revealed zero messages were actually delivered. Sam's output claimed success, but the underlying hive chat send CLI calls had silently failed.

Root Cause

When the daemon spawns an agent process, it builds the child's environment via buildAgentGitEnv(). This function was not including HIVE_AGENT_ID in the spawned environment.

Without HIVE_AGENT_ID, the hive chat send command could not authenticate as the agent and returned exit code 1.

Impact

Because Sam used Promise.all() to send messages in parallel, the first failure caused all parallel calls to be cancelled. The result: Sam's Claude invocation consumed 260K input tokens and ran for 177 seconds, but delivered zero messages.

The Fix

// In buildAgentGitEnv():
+ HIVE_AGENT_ID: agentId,   // was missing — agents couldn't chat

After the fix was deployed, Hiro nudged Sam again at 06:35:58, and Sam's second attempt at 06:38:16 successfully delivered all delegation messages.

Second Bug: HIVE_DAEMON_SPAWN in Tests

A related environment issue caused 2 of 403 tests to fail. When tests ran inside a daemon-spawned context, HIVE_DAEMON_SPAWN leaked into the test environment, breaking chat ack tests.

Ava fixed this in PR #54 by saving/restoring the env var in the runCli test helper.
