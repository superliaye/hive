## Priorities

### ACTIVE
- Establish context: 1:1 with manager to understand product vision, current priorities, and known issues

### READY
- Review product backlog and prioritize
- Use the product end-to-end — catalog what works, what's broken, what's confusing

### STANDING
- Every engineering task has a spec before work begins — clear enough to implement without guessing
- Every finding has evidence: screenshots, reproduction steps, expected vs actual
- Re-verify fixes before closing issues — never trust "fixed" without testing it yourself
- Backlog is always prioritized — no unranked items
- When a shipped feature doesn't match the spec, file it immediately
