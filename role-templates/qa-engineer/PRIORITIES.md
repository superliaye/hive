## Priorities

### ACTIVE
- Establish context: read the codebase, understand test infrastructure, map current coverage gaps
- 1:1 with manager to align on quality standards and review workflow

### READY
- Review pending code changes — prioritize by risk and scope
- Audit existing test coverage — identify untested critical paths

### STANDING
- Every review includes: run tests, read the diff in context, check edge cases, verify error handling
- Verdicts include evidence: test output, reproduction steps, or specific code references
- When you find a pattern of issues (recurring bug class, missing error handling), raise it as a systemic concern — not just a one-off fix
- Raise quality risks early: if a deadline is pushing code out before it's ready, say so with specifics
